那么最后一个，就是开发时候我用了代理来进行跨域，可是我部署上线的时候疯狂报错，怎么办呢

这个问题，来自于我们执行了npm run dev,而我们上线部署执行的是npm run build,所以，代理操作就只能在dev下起作用了，那么如何解决呢

  • 第一种，就是在npm run build 之前手动将所有的url地址前的api删掉，很明显，这种方法在axios请求多的时候是不可取的

  • 第二种就是使用nginx反向代理，当然如何设置请自行百度，为什么没放配置方法？因为我用的第三种

  • 第三种就是在项目判断是dev环境还是build环境，然后进行区别api的值就ok啦，是不是特别简单呢
