static和assets文件夹有什么区别

看过很多网上资料，都描述的一样：static下的文件不会被打包，而assets会被webpack打包。那么，什么样的文件放在assets，什么样的文件放在static中呢？

  • static中放那些npm中下载不到的第三方包，和不需要被编译打包的images和css或者js，这个看项目

  • assets中放会被编译打包的文件，比如webapp底部导航栏的图片，公共的common.css文件等等

这个也很简单了./是相对路径，@/也是相对路径，但是起点是src文件夹，这个在build/webpack.base.conf.js中有设置

es6 import区别

  • import FunName from ‘../xxx’这个都知道，引入某个文件,文件里只有一个export default
  • import {xxx} from ‘../xxx’ 这个呢被引入的文件中有许多个export
  • import * as moment from ‘moment’; 这个呢是将moment文件中的所有变量都变成了一个名字为moment的对象，而moment文件中都是一个个变量
